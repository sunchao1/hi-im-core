## 4. 双项目面试口述提纲（约 30 分钟）

### A. 每日健康 IM（10 分钟 · IM 岗必讲）

1. **背景**：医患双端、SCRM 协同，为何自研而非第三方 SDK  
2. **协议**：Protobuf 消息体、会话模型、私聊/群聊路由  
3. **存储分层**：Redis 在线态 + MySQL 持久化；群成员与权限  
4. **与 hi-im-core 关系**：Go 业务层 + 热路径可下沉 C++ 总线（分工，非同一系统）  
5. **结果**：长期稳定、可扩展模块迭代  

### B. hi-im-core（12 分钟 · IM/C++ 岗必讲）

1. **问题**：进程间热路径要 **低延迟、高 recv/s**，Kafka 不适合毫秒级 fan-out  
2. **架构**：双平面 FORWARD/BACKEND、Bridge、Listener→Reactor→Worker→Distributor  
3. **协议**：20B bus wire v1、AUTH/SUB/KPALIVE  
4. **性能**：SPSC 有界队列、eventfd、TCP_NODELAY、**153,942 recv/s** 怎么测的  
5. **工程**：Docker Compose 一键验收、ctest、baseline JSON  

### C. K8s 上云交付（5 分钟 · IM 岗加分）

1. 服务容器化、镜像构建与多环境配置  
2. K8s Deployment/Service、配置与密钥管理  
3. IM 类有状态/长连接在 K8s 上的 **扩容与滚动** 注意点（按你真实经验讲，勿编造未做过项）  

### D. 乐视触达（3 分钟 · 时间不够时压缩）

- Push + Inbox 双通道；Fan-out + 分片；**10 分钟 → 1 分钟** 量化  

### E. 优信（仅 C++ / 金融岗 · 3 分钟）

- PHP 贷后推银行材料；大文件慢 → 引出 hi-file-transfer **设计**（实现后再讲 demo）
