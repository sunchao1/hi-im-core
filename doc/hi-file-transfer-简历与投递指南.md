## 1. 你为什么需要第二个 C++ 项目？

| 投递方向 | hi-im-core  alone | + hi-file-transfer |
|----------|-------------------|---------------------|
| IM / 消息 / 触达 / 基础架构 | ✅ 足够 | 可选加分 |
| **泛 C++ 后端 / 中间件 / 存储·网络** | ⚠️ 偏窄（小包总线） | ✅ **场景互补** |
| 金融 / 银行对接 / 文件网关 | ⚠️ 不对口 | ✅ **对口** |
| 纯游戏 / 图形 / 嵌入式 | ❌ 不对口 | ❌ 仍不对口 |

**结论：** 投 **C++ 岗** 时，**hi-im-core + hi-file-transfer（实现后）** 比单项目稳；但 **不能替代** 真实工作年限，且 **优信 PHP 经历仍要保留**。


## 4. 适合你的 C++ 岗位（按匹配度）

### 强烈推荐（主投）

| 岗位 | 匹配点 |
|------|--------|
| **C++ 后端 / 服务端开发** | 双项目：高并发 + 文件 IO |
| **基础架构 / 中间件 / 平台开发** | epoll、队列、分布式、Docker |
| **IM / 实时通信 / 长连接** | hi-im-core + 乐视/每日健康 |
| **网络库 / 网关 / 接入层** | Reactor、TCP、SFTP |
| **金融科技 · 银行/支付对接** | 优信 + hi-file-transfer 叙事 |

### 可以投（需补话术或弱项）

| 岗位 | 说明 |
|------|------|
| 存储 / 数据库内核 | 强调文件持久化、checkpoint，弱项分布式存储理论 |
| 音视频 / 流媒体 | 优信 **视频面签** 可作业务背景，需补 codec 知识 |
| 云原生 C++（K8s  Operator） | 有 Docker/K8s 技能可沾边 |

### 不建议主投

| 岗位 | 原因 |
|------|------|
| 游戏客户端 / UE / Unity | 栈无关 |
| 嵌入式 / 单片机 | 栈无关 |
| 纯量化 / HFT | 缺金融 C++ 低延迟实战 |
| 算法岗 | 年龄与方向 |

## 5. 「两个 C++ 项目够吗？」

**够用来拿到 C++ 面试**，尤其 **39 岁 + 16 年** 走的是 **资深/专家** 通道，面试官更看重：

1. **能不能讲清** 架构、瓶颈、为什么不用另一种方案  
2. **有没有量化**（hi-im-core 15 万 msg/s；hi-file-transfer 待 MB/s + 断点 demo）  
3. **真实大厂/IM/金融** 履历（乐视、每日健康、优信）  

两个 C++ 项目 **解决「只有一个小 demo」的问题**，不能解决：

- 无 **生产 C++ 年限** 的标签 → 用「C++ 个人项目 + Go/PHP 线上大规模」组合讲  
- 泛投 **所有 C++ JD** → 仍要筛岗位（见上表）  

**务实预期（上海）：**

- 双 C++ 项目 + 16 年：**C++ 岗 40～55K×14** 可谈  
- 仅 IM 岗：**45～60K** 可能更高（主栈 Go 更贴 JD）
