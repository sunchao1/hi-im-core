### hi-im-core · 高并发 IM 进程消息总线（C++20）

**独立设计与实现 · 2025 — 2026**

**技术栈**：C++20 · epoll · CMake · Docker Compose · TCP 二进制协议

**内容**

1. 净室设计并实现双平面 Hub（FORWARD / BACKEND）与 Bridge：FORWARD 上行业务 publish 至 BACKEND，BACKEND 下行业务按 IM 头 nid async_send 至 FORWARD。
2. 实现 Listener → Reactor → Worker → Distributor 流水线，有界 SPSC 环、eventfd 唤醒、Snap 拼帧；20 字节 bus wire v1 协议（AUTH / SUB / KPALIVE），配套单测与端到端冒烟。
3. 提供 hi-im-bench 压测与 Docker Compose 一键 build / accept / bench 验收链路。

**业绩**

1. M1 publish 压测 **recv/s 153,942**（256B payload、10s、单 Publisher），超过目标 140,000 msg/s；基线见仓库 `bench/baseline-v1.json`。
2. 与 Go 业务层、Kafka 冷路径分工清晰：core 负责毫秒级进程间热路径，Kafka 负责削峰持久化。
