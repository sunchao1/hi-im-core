## 四队列在源码中的锚点

| 队列 | 定义 | Push 位置 | Pop 位置 |
|------|------|-----------|----------|
| connq | `context_impl.cpp` 构造 | `listener.cpp:Run` | `reactor.cpp:DrainNewConnections` |
| recvq | `context_impl.cpp` 构造 | `reactor.cpp:EnqueueInbound` | `worker.cpp:Run` |
| distq | `context_impl.cpp` 构造 | `context_impl.cpp:Publish/AsyncSend` | `distributor.cpp:Run` |
| sendq | `context_impl.cpp` 构造 | `distributor.cpp:RouteToSendQueue` | `reactor.cpp:DrainSendQueue` |

## 文件头注释格式说明

每个已注释文件顶部有类似块：

```cpp
// =============================================================================
// 文件：xxx.cpp
// 职责：...
// 流水线角色：Listener → Reactor → Worker → Distributor 的 xxx 层。
// 涉及队列：...
// 执行线程：...
// =============================================================================
```

方法注释用 `//` 单行或 `///` Doxygen 风格，说明：

- **干什么**
- **谁调用 / 在哪个线程执行**
- **与哪条队列或哪张路由表相关**
